Google Search Indexing Checklist

Purpose
This file is the practical next-step checklist for getting QuizMintAI indexed in Google after the SEO migration work.

This is not a strategy document.
This is the execution checklist for Google Search Console, sitemap submission, URL inspection, and early indexing follow-up.

Step 2 | Add the site to Google Search Console

Recommended property type
- use a Domain property if possible
- if Domain property is not possible, use URL-prefix property for:
  - https://www.quizmintai.com

How to verify
- best option:
  - DNS TXT record verification
- other possible methods:
  - HTML file upload
  - HTML tag
  - Google Analytics
  - Google Tag Manager

Best practice
- verify the full domain if possible so all subdomains and protocol variations are covered

How to inspect a URL
1. paste the full URL into Search Console URL Inspection
2. wait for the result
3. if the page is not indexed, click:
   - Request Indexing
4. repeat for the next priority page

Notes
- do not panic if Google says the URL is not yet on Google at first
- request indexing for priority pages first, not every URL blindly at once

Step 5 | Check indexing blockers

If a page does not index, review:
- canonical points to the right URL
- page is not marked noindex
- robots.txt is not blocking it
- page is in the sitemap
- page has internal links from important pages
- page content is not too thin or too similar to another page

Common Search Console statuses to watch
- Discovered - currently not indexed
- Crawled - currently not indexed
- Duplicate without user-selected canonical
- Alternate page with proper canonical tag
- Excluded by noindex tag
- Blocked by robots.txt

Step 6 | Submit pages in a sensible order

Recommended order
1. homepage
2. resources page
3. 3 category pages
4. core workflow pages
5. main generator pages
6. segment and regional pages
7. subject pages
8. blog hub
9. blog posts

Why this order
- Google should first understand the main site structure and category-level pages
- deeper and narrower pages are easier to understand after the main architecture is crawled

Step 8 | Watch the Performance report

Track
- impressions
- clicks
- CTR
- average position

Break observations by
- homepage
- category pages
- workflow pages
- generator pages
- segment pages
- regional pages
- subject pages

What to look for
- pages with impressions but low CTR
- pages getting indexed but not attracting clicks
- pages attracting the wrong queries
- pages with strong impressions that deserve deeper content improvement

Step 9 | Improve pages after first indexing

If impressions appear but clicks are weak
- improve title
- improve meta description
- improve H1 clarity
- improve internal links from homepage, resources, and related pages

If Google ignores a page
- make the page more distinct
- add stronger internal links
- reduce overlap with similar pages
- ensure the page solves a clearly different user need

Step 10 | Keep a simple weekly indexing routine

Weekly checklist
- check sitemap status
- inspect newly added public pages
- review Pages report
- review Performance report
- note which pages gained impressions
- note which pages still are not indexed
- improve 1 to 3 pages each week based on live data

Important reminders
- indexing is not instant
- requesting indexing does not guarantee indexing
- sitemap submission helps discovery, not ranking by itself
- internal linking and page quality still matter
- category and workflow pages should be prioritized over lower-value long-tail pages

Simple success signs
- sitemap accepted
- homepage indexed
- category pages indexed
- workflow pages begin getting impressions
- generator pages start supporting broader workflow discovery
- Search Console begins showing real query data for the new public pages

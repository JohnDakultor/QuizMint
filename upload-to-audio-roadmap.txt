QuizMintAI Upload-To-Audio Roadmap

Purpose
- let teachers upload a book chapter, PDF, lesson handout, or similar file
- generate student-ready narrated audio from that uploaded material
- keep the feature aligned with teacher workflow, student sharing, and accessibility support

Core Product Idea
- teacher uploads a file
- QuizMintAI extracts and cleans the text
- teacher chooses the narration style
- QuizMintAI generates audio
- teacher previews the audio and readable content
- teacher shares the audio and visible reading content to students

Recommended Positioning
- "Create student-ready audio learning material from teacher files"
- "Turn lesson PDFs and reading handouts into narrated study audio"
- "Support listening-based review, remediation, and reading accessibility"
- "Present uploaded teaching material in a read-aloud classroom mode"

Best Early Use Cases
- reading support for struggling students
- narrated review materials before quizzes
- listening versions of teacher-created lesson notes
- homework companion audio
- remediation and follow-up support
- student access support for long reading passages
- teacher-led classroom read-aloud on a shared screen
- presentable digital reading mode for uploaded book chapters and PDFs

Important Constraints
- start with teacher-owned, school-created, or teacher-uploaded instructional material
- be careful with copyrighted commercial books
- do not market it as a way to mass-convert copyrighted books into downloadable audiobooks
- very large files must be processed through staged async chunking, not one-shot extraction

Large-File Handling Strategy

Why This Matters
- some teachers may upload very large PDFs, books, or compiled materials
- files in the thousands of pages should not be processed in one direct request
- large-file support requires a different pipeline than normal classroom handouts

Recommended Handling
- ingest the file once
- extract text in page or chunk batches
- store intermediate extraction artifacts
- divide content into chapters or sections
- generate audio per section
- expose the output as:
  - multi-part audio
  - chapter playlist
  - summary-first audio

What Not To Do
- do not load the entire file into memory at once
- do not try to generate one giant audio file in a single synchronous request
- do not make the browser wait for full completion

Large-File Product Rules
- v1 should define practical file-size and page-count limits
- if a file exceeds the fast-path threshold:
  - move it into a long-running async job
  - show staged progress
  - return segmented output instead of one monolithic output
- for extremely large uploads, offer:
  - summary narration
  - selected chapters only
  - teacher-selected page ranges

Large-File Engineering Tasks
- add page-count / size detection before processing
- add chunked extraction jobs
- persist extraction checkpoints
- support resumable audio generation per chunk
- support partial completion instead of all-or-nothing failure
- support chapterized storage and playback

Pricing Recommendation
- Pro/Premium candidate feature
- recommended rollout:
  - Pro: summary narration or limited monthly audio generation
  - Premium: full narration, longer files, richer voices, student sharing

Security And Policy Notes
- scope access to teacher-owned uploads only
- respect current account and class boundaries
- add clear policy language for copyrighted materials
- consider disabling direct full-book conversion for obvious commercial copyrighted books if needed
